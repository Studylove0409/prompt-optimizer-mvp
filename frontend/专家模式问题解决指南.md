## 🔍 问题诊断步骤

### 1. 打开调试工具
访问 `debug-expert.html` 页面进行详细诊断：
```
https://你的域名/debug-expert.html
```

### 2. 检查浏览器控制台
按 `F12` 打开开发者工具，查看 Console 标签页：

**正常情况下应该看到：**
```
准备初始化专家访谈管理器...
Document readyState: complete
开始创建ExpertInterviewManager实例...
开始初始化专家访谈DOM元素...
✅ 专家访谈DOM元素初始化完成
✅ 专家访谈管理器初始化成功
window.expertInterviewManager: object
```

**如果看到错误信息，请继续下面的步骤。**

### 3. 常见错误及解决方案

#### 错误类型1: DOM元素缺失
```
缺失专家访谈必要的DOM元素: expertInterviewSection, analysisStage...
```

**解决方案：**
- 确保 `index.html` 包含完整的专家访谈区域代码
- 检查是否在某次更新中意外删除了相关HTML结构

#### 错误类型2: 脚本加载失败
```
ExpertInterviewManager is not defined
```

**解决方案：**
- 检查 `expert-interview.js` 是否正确加载
- 确认文件路径和文件名是否正确
- 检查服务器是否正确提供JavaScript文件

#### 错误类型3: 初始化时机问题
```
Cannot read property 'getElementById' of null
```

**解决方案：**
- 确保脚本在DOM加载完成后执行
- 检查脚本标签的位置（应该在 `</body>` 标签前）

## 🛠️ 快速修复方法

### 方法1: 手动重新初始化
在浏览器控制台中执行：
```javascript
// 检查管理器状态
console.log('专家访谈管理器:', window.expertInterviewManager);

// 强制重新初始化
if (window.forceReinitializeExpertInterview) {
    window.forceReinitializeExpertInterview();
}

// 验证结果
console.log('重新初始化后:', !!window.expertInterviewManager);
```

### 方法2: 检查DOM元素完整性
```javascript
// 检查必要的DOM元素
const requiredElements = [
    'expertInterviewSection',
    'analysisStage', 
    'interviewStage',
    'synthesisStage'
];

requiredElements.forEach(id => {
    const element = document.getElementById(id);
    console.log(`${id}:`, !!element);
});
```

### 方法3: 强制加载脚本
如果脚本未加载，在控制台执行：
```javascript
// 动态加载专家访谈脚本
const script = document.createElement('script');
script.src = 'expert-interview.js';
script.onload = () => console.log('专家访谈脚本重新加载完成');
document.head.appendChild(script);
```

## 🔧 Vercel部署特殊注意事项

### 1. 文件大小写敏感
确保所有文件名大小写正确，Linux服务器区分大小写。

### 2. 文件路径检查
```javascript
// 在控制台检查文件是否可访问
fetch('./expert-interview.js')
  .then(response => console.log('专家访谈脚本状态:', response.status))
  .catch(error => console.error('无法加载专家访谈脚本:', error));
```

### 3. 缓存问题
如果更新后仍有问题，尝试强制刷新：
- `Ctrl + F5` (Windows)
- `Cmd + Shift + R` (Mac)
