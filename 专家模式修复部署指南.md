# 专家模式修复部署指南

## 🎯 问题诊断结果

**根本原因确认：**
1. ✅ **后端缺少expert模式支持** - 在 `backend/app/constants.py` 中 `get_prompt_template_by_mode()` 函数缺少expert模式处理
2. ✅ **前端初始化问题** - 专家访谈管理器可能存在时序初始化问题

## 🔧 已实施的修复

### 后端修复
1. **修改了 `backend/app/constants.py`**
   ```python
   def get_prompt_template_by_mode(mode: str) -> str:
       # ... 其他模式 ...
       elif mode == "expert":
           # expert模式直接返回用户原始输入，由前端的智能访谈系统处理
           return "{user_input_prompt}"
       else:  # general mode
           return META_PROMPT_TEMPLATE_GEMINI
   ```

2. **更新了 `backend/app/models.py`**
   ```python
   mode: str = "general"  # 可选值: general, business, drawing, academic, expert
   ```

### 前端修复
1. **增强了专家访谈管理器初始化逻辑**
   - 多重初始化策略
   - DOM加载状态检查
   - 详细的调试日志
   - 自动重试机制

2. **改进了API调用逻辑**
   - 智能检测管理器状态
   - 自动重新初始化
   - 增强的错误处理

3. **创建了调试工具**
   - `debug-expert.html` - 完整诊断界面
   - `test-expert.html` - 功能测试页面

```bash

  .then(r => console.log('API脚本状态:', r.status))
```

#### 方法3: 检查脚本加载顺序
确认 `index.html` 中的脚本标签顺序：
```html
<script src="utils.js"></script>
<script src="api.js"></script>
<!-- 其他脚本... -->
<script src="expert-interview.js"></script>  <!-- 必须在api.js之后 -->

```
