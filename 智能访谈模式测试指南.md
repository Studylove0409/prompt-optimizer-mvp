# 智能访谈模式测试指南

## 功能概述

智能访谈模式是一个三阶段的AI辅助提示词优化工作流：

1. **需求分析阶段** - AI分析用户的初步想法，识别缺失的关键信息
2. **智能访谈阶段** - 动态生成结构化问题，用户补充完善信息
3. **提示词合成阶段** - AI将所有信息融合生成专家级提示词

## 技术实现总结

### 后端API端点

- `POST /api/analyze` - 需求分析API (分析器)
- `POST /api/synthesize` - 提示词合成API (合成器)
- 现有的 `POST /api/optimize` - 保持兼容常规优化模式

### 数据模型

```python
# 分析请求/响应
AnalyzeRequest: {original_idea, model}
AnalyzeResponse: {questions[], summary, model_used}

# 合成请求/响应  
SynthesizeRequest: {original_idea, user_answers, model}
SynthesizeResponse: {optimized_prompt, model_used}
```

### 前端实现

- `ExpertInterviewManager` 类管理整个工作流
- 完整的HTML结构支持三阶段界面
- 集成到主优化流程，检测"expert"模式自动启动

## 测试流程

### 1. 启动测试

1. 打开应用首页
2. 在输入框中输入一个模糊的想法，例如："帮我写个市场分析报告"
3. 选择"思考专家"模式（标有"限时免费"标签）
4. 点击优化按钮

**预期行为：**
- 界面切换到智能访谈区域

**预期行为：**

**验证点：**
- [ ] 表单动态生成正确

### 4. 第三阶段测试 - 提示词合成

**预期行为：**
- 用户提交表单后进入第三阶段
- 显示"正在合成专业提示词..."

- 调用 `/api/synthesize` 端点

**验证点：**
- [ ] 合成器API正确调用

## 问题排查指南

### 🚨 常见问题及解决方案

#### 1. 智能访谈模式无法启动
**症状：** 点击优化按钮后没有反应或显示错误

**排查步骤：**
1. 打开浏览器开发者工具 (F12)
2. 检查控制台是否有JavaScript错误
3. 确认是否选择了"思考专家"模式
4. 检查网络请求是否发出

**可能原因：**
- 前端JavaScript加载失败
- DOM元素未正确初始化
- 模式选择器状态异常

#### 2. 第一阶段（分析）失败
**症状：** 显示"分析失败"或"网络连接失败"

**排查步骤：**
1. 检查控制台的详细错误信息
2. 验证分析器API是否可用：`POST /api/analyze`
3. 检查LLM模型响应格式

**调试命令：**
```bash
# 测试分析器API
curl -X POST http://localhost:8000/api/analyze \
  -H "Content-Type: application/json" \
  -d '{"original_idea": "帮我写个市场分析报告", "model": "gemini-2.0-flash"}'
```

**常见错误：**
- HTTP 500: LLM响应JSON解析失败
- HTTP 400: 请求参数错误
- 网络错误: 后端服务未启动

#### 3. JSON解析失败
**症状：** 后端日志显示JSON解析错误

**解决方案：**
- 已实现多重JSON解析策略
- 自动回退到备用问题
- 检查LLM模型是否稳定

#### 4. 第三阶段（合成）失败
**症状：** 访谈完成后合成失败

**排查步骤：**
1. 检查用户答案是否正确传递
2. 验证合成器API：`POST /api/synthesize`
3. 查看后端合成日志
