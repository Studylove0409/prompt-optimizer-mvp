# 智能访谈模式测试指南

## 功能概述

智能访谈模式是一个三阶段的AI辅助提示词优化工作流：

1. **需求分析阶段** - AI分析用户的初步想法，识别缺失的关键信息
2. **智能访谈阶段** - 动态生成结构化问题，用户补充完善信息
3. **提示词合成阶段** - AI将所有信息融合生成专家级提示词

## 技术实现总结

### 后端API端点

- `POST /api/analyze` - 需求分析API (分析器)
- `POST /api/synthesize` - 提示词合成API (合成器)
- 现有的 `POST /api/optimize` - 保持兼容常规优化模式

### 数据模型

```python
# 分析请求/响应
AnalyzeRequest: {original_idea, model}
AnalyzeResponse: {questions[], summary, model_used}

# 合成请求/响应  
SynthesizeRequest: {original_idea, user_answers, model}
SynthesizeResponse: {optimized_prompt, model_used}
```

### 前端实现

- `ExpertInterviewManager` 类管理整个工作流
- 完整的HTML结构支持三阶段界面
- 集成到主优化流程，检测"expert"模式自动启动

## 测试流程

### 1. 启动测试

1. 打开应用首页
2. 在输入框中输入一个模糊的想法，例如："帮我写个市场分析报告"
3. 选择"思考专家"模式（标有"限时免费"标签）
4. 点击优化按钮

### 2. 第一阶段测试 - 需求分析

**预期行为：**
- 界面切换到智能访谈区域
- 显示"需求分析"阶段，状态为"正在分析您的想法..."
- 进度条显示33%（1/3）
- 调用 `/api/analyze` 端点

**验证点：**
- [ ] 分析器API正确调用
- [ ] 返回JSON格式的问题列表
- [ ] 问题数量在3-6个之间
- [ ] 每个问题包含key, question, type, placeholder, required字段

### 3. 第二阶段测试 - 智能访谈

**预期行为：**
- 自动进入第二阶段
- 根据API返回的问题动态生成表单
- 进度条显示66%（2/3）
- 必填项标记红色星号

**验证点：**
- [ ] 表单动态生成正确
- [ ] 不同类型的input正确渲染（text/textarea）
- [ ] 必填验证工作正常
- [ ] "返回修改"和"提交并生成"按钮可用

### 4. 第三阶段测试 - 提示词合成

**预期行为：**
- 用户提交表单后进入第三阶段
- 显示"正在合成专业提示词..."
- 进度条显示100%（3/3）
- 调用 `/api/synthesize` 端点

**验证点：**
- [ ] 合成器API正确调用
- [ ] 用户答案正确传递
- [ ] 生成高质量的专业提示词
- [ ] 结果可以复制
- [ ] "重新开始"功能正常

## API测试用例

### 分析器API测试
```bash
curl -X POST http://localhost:8000/api/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "original_idea": "帮我写个市场分析报告",
    "model": "gemini-2.0-flash"
  }'
```

### 合成器API测试
```bash
curl -X POST http://localhost:8000/api/synthesize \
  -H "Content-Type: application/json" \
  -d '{
    "original_idea": "帮我写个市场分析报告",
    "user_answers": {
      "product_or_service": "智能手机",
      "target_market": "中国一线城市的25-35岁白领群体",
      "analysis_focus": "竞争对手分析、市场规模、用户需求"
    },
    "model": "gemini-2.0-flash"
  }'
```

## 错误处理测试

### 网络错误
- [ ] API调用失败时显示友好错误信息
- [ ] 用户可以重试操作

### 数据格式错误
- [ ] JSON解析失败时使用备用问题
- [ ] 分析器返回格式不正确时优雅降级

### 用户交互错误
- [ ] 必填项未填写时阻止提交
- [ ] 无效输入时提供清晰提示

## 性能测试

- [ ] 分析阶段响应时间 < 10秒
- [ ] 合成阶段响应时间 < 15秒
- [ ] 界面切换流畅无卡顿
- [ ] 大量文本输入处理正常

## 兼容性测试

- [ ] 桌面端浏览器正常工作
- [ ] 移动端响应式布局适配
- [ ] 不同AI模型都能正常工作

## 已知限制

1. 目前仅支持text和textarea类型的问题
2. 合成阶段无法中途取消
3. 历史记录保存为"expert"模式

## 未来优化方向

1. 添加select类型问题支持
2. 实现进度保存和恢复
3. 增加模板化问题库
4. 支持多轮访谈
